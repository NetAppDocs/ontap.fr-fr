= SnapMirror Sauvegardes cloud vers un stockage objet
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
_SnapMirror Cloud_ est une technologie de sauvegarde et de restauration conçue pour les utilisateurs d'ONTAP qui souhaitent faire migrer leurs flux de travail de protection des données vers le cloud ou vers une solution de stockage objet sur site. Les entreprises qui abandonnent les architectures existantes de sauvegarde sur bande peuvent utiliser le stockage objet comme référentiel alternatif pour la conservation des données à long terme et l'archivage. SnapMirror Cloud fournit une réplication d'ONTAP vers le stockage objet dans le cadre d'une stratégie de sauvegarde incrémentielle infinie.

SnapMirror Cloud est une fonctionnalité ONTAP sous licence. SnapMirror Cloud a été introduit dans ONTAP 9.8 comme une extension de la famille des technologies de réplication SnapMirror. Alors que SnapMirror est fréquemment utilisé pour les sauvegardes ONTAP à ONTAP, SnapMirror Cloud utilise le même moteur de réplication pour transférer les instantanés ONTAP vers des sauvegardes de stockage objet compatibles S3.

Conçu pour les cas d'utilisation de sauvegarde, SnapMirror Cloud prend en charge la conservation à long terme et les flux de travail d'archivage. Comme avec SnapMirror, la sauvegarde SnapMirror Cloud initiale effectue un transfert de base d'un volume. Pour les sauvegardes suivantes, SnapMirror Cloud génère un instantané du volume source et transfère l'instantané, contenant uniquement les blocs de données modifiés, vers une cible de stockage objet.

SnapMirror Cloud permet de configurer des relations entre les systèmes ONTAP et certaines cibles de stockage d’objets sur site et dans le cloud public, notamment Amazon S3, Google Cloud Storage et Microsoft Azure Blob Storage. Les autres cibles de stockage d’objets sur site incluent StorageGRID et ONTAP S3.

Outre l'utilisation de ONTAP System Manager pour gérer les configurations SnapMirror Cloud, plusieurs options d'orchestration sont disponibles pour gérer les sauvegardes SnapMirror Cloud :

* Plusieurs partenaires de sauvegarde tiers proposent une assistance pour la réplication SnapMirror Cloud. Les fournisseurs participants sont disponibles sur le link:https://www.netapp.com/blog/new-backup-architecture-snapdiff-v3/["Blog NetApp"^].
* Sauvegarde et restauration NetApp pour une solution native NetApp pour les environnements ONTAP
* API pour développer des logiciels personnalisés pour les workflows de protection des données ou exploiter les outils d'automatisation



NOTE: SnapMirror Cloud, gérée par System Manager, prend uniquement en charge StorageGrid et ONTAP S3 comme destination de stockage d’objets. Les autres fournisseurs ne sont pas pris en charge.


NOTE: L'utilisation des API Cloud SnapMirror est généralement réservée aux partenaires de sauvegarde. Les clients qui souhaitent développer leurs propres solutions basées sur les API doivent contacter leur équipe commerciale NetApp pour obtenir des informations sur les conditions de licence.
