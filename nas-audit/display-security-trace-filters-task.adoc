= Afficher des informations sur les filtres de trace de sécurité dans les SVM ONTAP
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Consultez les détails du filtre de suivi de sécurité sur votre storage virtual machine (SVM) pour identifier les événements d'accès que chaque filtre trace.

[role="tabbed-block"]
====
.System Manager
--
À partir d'ONTAP 9.6, vous pouvez utiliser System Manager pour tracer l'accès aux fichiers sur le SVM.

.Étapes
. Ouvrez le tableau de bord System Manager.
. Sous *Cluster*, sélectionnez *Storage VMs*.
. Sélectionnez image:icon_kabob.gif["Icône des options de menu"] pour le SVM applicable.
. Sélectionnez *Trace file access*.
. Définissez le type de protocole d'accès aux fichiers sur *SMB/CIFS* ou *NFS*.
. Saisissez le *nom d'utilisateur*.
. Saisissez l'adresse IP du client.
. Facultativement, sélectionnez une ou plusieurs des options suivantes :
+
.. Sélectionnez *Trace particular path* pour spécifier le fichier ou le chemin d'accès au fichier à tracer.
.. Sélectionnez *Afficher uniquement les entrées d'accès refusé* pour limiter la trace aux événements d'accès refusé.


. Sélectionnez *Start tracing*.


--
.CLI
--
Utilisez l'interface de ligne de commande (CLI) pour tracer l'accès aux fichiers sur la SVM.

.Étape
. Affiche des informations sur les entrées du filtre de trace de sécurité à l'aide de `vserver security trace filter show` commande.
+
Pour en savoir plus, `vserver security trace filter show` consultez le link:https://docs.netapp.com/us-en/ontap-cli/vserver-security-trace-filter-show.html["Référence de commande ONTAP"^].



.Exemples
La commande suivante affiche des informations sur tous les filtres de trace de sécurité sur le SVM vs1 :

[listing]
----
cluster1::> vserver security trace filter show -vserver vs1
Vserver  Index   Client-IP            Path            Trace-Allow  Windows-Name
-------- -----  -----------  ----------------------   -----------  -------------
vs1        1    -            /dir1/dir2/file.txt          yes      -
vs1        2    -            /dir3/dir4/                  no       mydomain\joe
----
