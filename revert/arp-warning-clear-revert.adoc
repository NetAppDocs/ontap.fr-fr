= Résolvez les alertes d'activité dans la protection autonome contre les ransomwares (ARP) avant une restauration ONTAP.
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Avant de revenir à ONTAP 9.17.1 ou à une version antérieure, vous devez répondre à tous les avertissements d'activité anormale signalés par la protection autonome contre les ransomwares (ARP) et supprimer toutes les captures d'écran ARP associées.

.Avant de commencer
Vous devez disposer de privilèges « avancés » pour supprimer les instantanés ARP.

.Étapes
. Répondez aux avertissements d'activité anormale signalés par link:../anti-ransomware/respond-abnormal-task.html["ARP"] et résolvez tout problème potentiel.
. Veuillez confirmer la résolution de ces problèmes avant de revenir en arrière en sélectionnant *Mettre à jour et effacer les types de fichiers suspects* pour enregistrer votre décision et reprendre la surveillance ARP normale.
. Répertoriez les captures d'écran ARP associées aux avertissements en exécutant la commande suivante :
+
[source, cli]
----
volume snapshot snapshot show -fs-version 9.18
----
. Supprimez toutes les captures d'écran ARP associées aux avertissements :
+

NOTE: Cette commande supprime tous les instantanés ayant le format utilisé par la version actuelle d' ONTAP, potentiellement pas seulement les instantanés ARP.  Assurez-vous d'avoir pris toutes les mesures nécessaires pour tous les instantanés qui seront supprimés avant d'exécuter cette commande.

+
[source, cli]
----
volume snapshot prepare-for-revert -node <node_name>
----
