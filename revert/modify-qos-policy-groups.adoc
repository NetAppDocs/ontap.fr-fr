= Modifiez les groupes de stratégies QoS créés dans ONTAP 9.19.1 ou une version ultérieure avant de revenir depuis ONTAP 9.19.1
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Les groupes de stratégies QoS adaptatifs et non adaptatifs sont activés automatiquement pour augmenter temporairement le plafond de débit d’une charge de travail de 50 % pendant 1 seconde afin de répondre par défaut aux demandes à court terme de performances accrues. À partir de ONTAP 9.19.1, vous pouvez modifier ces paramètres par défaut. Dans ONTAP 9.18.1 et les versions antérieures, ces paramètres ne peuvent pas être modifiés. Si vous avez modifié les paramètres par défaut de tout groupe de stratégies QoS créé dans ONTAP 9.19.1 ou une version ultérieure, vous devez les réinitialiser avant de pouvoir revenir de ONTAP 9.19.1 à ONTAP 9.18.1.

.Étapes
. Vérifiez si vous avez activé des groupes de stratégies QoS pour l'augmentation automatique du plafond de débit :
+
[source, cli]
----
qos policy-group show – instance
----
. Si des groupes de stratégies sont configurés avec `burst-percent`, `burst-iops` et `burst-duration`, définissez ces options sur 0.
+
[source, cli]
----
qos policy-group modify -policy-group <policy-group> -burst-percent 0 -burst-iops 0 -burst-duration 0
----
