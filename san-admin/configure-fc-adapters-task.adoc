= Configurez les adaptateurs FC
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Les adaptateurs FC intégrés et certaines cartes d'adaptateur d'extension FC peuvent être configurés individuellement en tant qu'initiateurs ou ports cibles. Les autres adaptateurs d'extension FC sont configurés en usine en tant qu'initiateurs ou cibles et ne peuvent pas être modifiés. Des ports FC supplémentaires sont également disponibles via les cartes UTA2 prises en charge configurées avec les adaptateurs FC SFP+.

Les ports initiateurs peuvent être utilisés pour se connecter directement aux tiroirs disques back-end, et éventuellement à des baies de stockage étrangères. Les ports cibles peuvent être utilisés pour la connexion uniquement aux commutateurs FC.

Le nombre de ports intégrés et de ports CNA/UTA2 configurés pour FC varie en fonction du modèle du contrôleur. Les adaptateurs d'extension de cible pris en charge varient également en fonction du modèle de contrôleur. Reportez-vous à la link:https://hwu.netapp.com["NetApp Hardware Universe"^] pour obtenir la liste complète des ports FC intégrés et des adaptateurs d'extension cible pris en charge pour votre modèle de contrôleur.



[[configure-fc-adapters-for-initiator-mode]]
== Configurer les adaptateurs FC pour le mode initiateur

Le mode initiateur est utilisé pour connecter les ports à des lecteurs de bande, des bibliothèques de bandes ou un stockage tiers avec link:https://docs.netapp.com/us-en/ontap-fli/get-started/learn-about-san-data-migration.html["Importation de LUN étrangers (FLI)"^].

.Avant de commencer
* Les LIF présentes sur l'adaptateur doivent être supprimées de n'importe quel ensemble de ports dont elles sont membres.
* Toutes les LIF de chaque machine virtuelle de stockage (SVM) utilisant le port physique à modifier doivent être migrées ou détruites avant de changer la personnalité du port physique de cible à initiateur.


[NOTE]
====
NVMe/FC ne prend pas en charge le mode initiateur.

====
.Étapes
. Supprimer toutes les LIFs de l'adaptateur :
+
[source, cli]
----
network interface delete -vserver _SVM_name_ -lif _lif_name_,_lif_name_
----
. Mettez votre adaptateur hors ligne :
+
[source, cli]
----
network fcp adapter modify -node _node_name_ -adapter _adapter_port_ -status-admin down
----
+
Si l'adaptateur ne passe pas hors ligne, vous pouvez également retirer le câble du port d'adaptateur approprié du système.

. Modifiez l'adaptateur de la cible à l'initiateur :
+
[source, cli]
----
system hardware unified-connect modify -t initiator _adapter_port_
----
. Redémarrez le nœud hébergeant l'adaptateur que vous avez changé.
. Vérifier que les ports FC sont configurés dans l'état approprié pour votre configuration :
+
[source, cli]
----
system hardware unified-connect show
----
. Remettre la carte en ligne :
+
[source, cli]
----
node run -node _node_name_ storage enable adapter _adapter_port_
----




[[configure-fc-adapters-for-target-mode]]
== Configurer les adaptateurs FC pour le mode cible

Le mode cible est utilisé pour connecter les ports aux initiateurs FC.

Les mêmes étapes sont utilisées pour configurer les adaptateurs FC pour le protocole FC et le protocole FC-NVMe. Cependant, seuls certains adaptateurs FC prennent en charge la connectivité FC-NVMe. Consultez la link:https://hwu.netapp.com["NetApp Hardware Universe"^] pour obtenir la liste des adaptateurs qui prennent en charge le protocole FC-NVMe.

.Étapes
. Mettez l'adaptateur hors ligne :
+
[source, cli]
----
node run -node _node_name_ storage disable adapter _adapter_name_
----
+
Si l'adaptateur ne passe pas hors ligne, vous pouvez également retirer le câble du port d'adaptateur approprié du système.

. Modifiez l'adaptateur de l'initiateur sur la cible :
+
[source, cli]
----
system node hardware unified-connect modify -t target -node _node_name_ adapter _adapter_name_
----
. Redémarrez le nœud hébergeant l'adaptateur que vous avez changé.
. Vérifiez que la configuration du port cible est correcte :
+
[source, cli]
----
network fcp adapter show -node _node_name_
----
. Mettez votre adaptateur en ligne :
+
[source, cli]
----
network fcp adapter modify -node _node_name_ -adapter _adapter_port_ -state up
----




[[configure-fc-adapters-for-direct-attachment-to-fc-and-fc-nvme-hosts]]
== Configurer les adaptateurs FC pour une connexion directe aux hôtes FC et FC-NVMe

À partir d'ONTAP 9.19.1, un commutateur FC n'est pas requis pour établir des connexions entre les hôtes FC et FC-NVMe et le stockage ONTAP. Les hôtes peuvent être connectés directement aux ports d'adaptateur FC sur les systèmes AFF, ASA, ASA r2 et FAS. Cette capacité de connexion directe peut s'avérer particulièrement précieuse sur les sites distants et dans les environnements hybrides où la simplicité est essentielle. Vous devez mettre le port FC hors ligne et modifier sa topologie avant qu'il puisse être connecté directement à un hôte.

Les ports FC utilisés dans les configurations de connexion directe présentent les restrictions suivantes :

* Le port ne peut être attribué qu'à une seule machine virtuelle de stockage (SVM).
* Une seule interface réseau (LIF) peut être attribuée au port après sa configuration pour la connexion directe.
* Le port peut prendre en charge le protocole FC ou le protocole FC-NVMe, mais pas les deux.


.Avant de commencer
Vous devez être un administrateur de cluster.

[role="tabbed-block"]
====
.System Manager
--
. Dans le Gestionnaire système, sélectionnez *Réseau > Ports FC*.
. Sélectionnez image:icon_kabob.gif["icône de sélection"] à côté du port que vous souhaitez configurer pour une connexion directe, puis sélectionnez *Modifier la topologie*.
. Sous *Topologie*, sélectionnez *Direct* puis sélectionnez *Je souhaite modifier la topologie et mettre le port hors ligne*.
. Sélectionnez *Enregistrer.*
. Connectez le câble du port cible directement à l'initiateur hôte.
. Si vous disposez d'un AFF, ASA ou système FAS, suivez ces étapes pour créer un LIF : Si vous disposez d'un système ASA r2, suivez link:https://docs.netapp.com/us-en/asa-r2/administer/manage-client-vm-access.html#create-a-lif-network-interface["cette procédure System Manager"^] pour créer un LIF.
+
.. Sélectionnez *Réseau > Interfaces* puis sélectionnez image:icon_add.gif["ajouter une icône"].
.. Sélectionnez le protocole FC ou NVMe/FC.
.. Nommez le LIF ou acceptez le nom généré par ONTAP.
.. Sélectionnez le port d'attache.
.. Sélectionnez *Enregistrer*.
+
Le port est automatiquement mis en ligne lors de la création de la LIF.





.Et la suite ?
Dans le Gestionnaire système, sélectionnez *Réseau > Ports FC* puis vérifiez que la *Topologie* est affichée comme *Direct*.

--
.CLI
--
. Vérifiez que le port que vous souhaitez utiliser pour la connexion directe prend en charge la topologie directe :
+
[source, cli]
----
network fcp adapter show -fields topologies-supported
----
. Consultez l'état administratif et opérationnel de l'interface LIF utilisée pour le port adaptateur :
+
[source, cli]
----
network interface show -vserver <SVM_name>
----
. Mettez hors ligne l'interface LIF utilisée pour le port adaptateur :
+
[source, cli]
----
network interface modify -vserver <SVM_name> -lif * -home-port <adapter_port> -status-admin down
----
. Retirez tous les LIF du port adaptateur :
+
[source, cli]
----
network interface delete -vserver <SVM_name> -lif * -home-port <adapter_port>
----
. Mettez l'adaptateur hors ligne :
+
[source, cli]
----
network fcp adapter modify -node <node_name> -adapter <adapter_port> -status-admin down
----
. Modifiez la topologie du port de l'adaptateur en direct :
+
[source, cli]
----
network fcp adapter modify -node <node_name> -adapter <adapter_port> -topology direct
----
. Connectez le câble du port cible directement à l'initiateur hôte.
. Créer une interface LIF unique :
+
[source, cli]
----
network interface create -vserver <SVM_name> -lif <lif_name> -data protocol fcp|fcp-nvme -home-node <node_name> -home-port <port_name>
----
+
Le port est automatiquement activé après la création de la LIF.

. Vérifiez que le port est configuré pour la connexion directe :
+
[source, cli]
----
network fcp interface show
----
+
La topologie doit être affichée comme directe.



--

[[configure-fc-adapter-speed]]
== Configuration de la vitesse de l'adaptateur FC

Vous devez configurer la vitesse du port cible de votre carte pour qu'elle corresponde à la vitesse du périphérique auquel elle se connecte, au lieu d'utiliser la négociation automatique. Un port défini pour la négociation automatique peut prendre plus de temps pour se reconnecter après un basculement/rétablissement ou une autre interruption.

.Description de la tâche
Cette tâche englobant tous les SVM (Storage Virtual machine) et toutes les LIFs d'un cluster, vous devez utiliser le `-home-port` et `-home-lif` paramètres pour limiter la portée de cette opération. Si vous n'utilisez pas ces paramètres, l'opération s'applique à toutes les LIFs du cluster, ce qui peut ne pas être souhaitable.

.Avant de commencer
Toutes les LIFs qui utilisent cet adaptateur comme port de home port doivent être hors ligne.

.Étapes
. Mettre hors ligne toutes les LIFs sur cet adaptateur :
+
[source, cli]
----
network interface modify -vserver * -lif * { -home-node node1 -home-port 0c } -status-admin down
----
. Mettez l'adaptateur hors ligne :
+
[source, cli]
----
network fcp adapter modify -node node1 -adapter 0c -state down
----
+
Si l'adaptateur ne passe pas hors ligne, vous pouvez également retirer le câble du port d'adaptateur approprié du système.

. Déterminez la vitesse maximale de l'adaptateur de port :
+
[source, cli]
----
fcp adapter show -instance
----
+
Vous ne pouvez pas modifier la vitesse de l'adaptateur au-delà de la vitesse maximale.

. Modifier la vitesse de l'adaptateur :
+
[source, cli]
----
network fcp adapter modify -node node1 -adapter 0c -speed 16
----
. Mettez la carte en ligne :
+
[source, cli]
----
network fcp adapter modify -node node1 -adapter 0c -state up
----
. Mettre en ligne toutes les LIFs sur l'adaptateur :
+
[source, cli]
----
network interface modify -vserver * -lif * { -home-node node1 -home-port 0c } -status-admin up
----
