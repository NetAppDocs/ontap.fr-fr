= Créer un instantané commun entre les groupes de cohérence ONTAP ou les SVM NAS
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
En plus des opérations de snapshot planifiées régulièrement, vous pouvez créer manuellement un link:../concepts/snapshot-copies-concept.html["snapshot"] commun entre les volumes du groupe de cohérence SnapMirror principal et les volumes du groupe de cohérence SnapMirror secondaire dans une configuration SAN ou au niveau SVM pour les configurations NAS.

.Description de la tâche
L'intervalle de création des instantanés planifiés est de 12 heures pour la politique Automated Failover et de 3 heures pour la politique Automated Failover Duplex.

.Avant de commencer
* La relation de groupe SnapMirror doit être en mode synchrone.


.Étapes
Vous pouvez créer un instantané commun à l'aide de System Manager ou de l'interface de ligne de commande ONTAP.

[role="tabbed-block"]
====
.System Manager
--
. Depuis le cluster de destination, accédez à *Protection > Réplication*.
. Localisez la source de la relation, sélectionnez image:icon_kabob.gif["Icône des options de menu"] et choisissez *Mettre à jour*.
. Choisissez *Conformément à la politique* ou *Sélectionner l'instantané* pour choisir un instantané existant.
. Sélectionnez *Mettre à jour*.


--
.CLI
--
. À partir du site de destination, créez un instantané commun :
+
`snapmirror update -destination-path <destination path>`

+
Exemple de configuration SAN :

+
[listing]
----
destination::>snapmirror update -destination-path vs1_dst:/cg/cg_dst
----
+
Exemple de configuration NAS :

+
[listing]
----
snapmirror update -destination-path vs_b:
----
. Surveiller la progression de la mise à jour :
+
`destination::>snapmirror show -fields newest-snapshot`
