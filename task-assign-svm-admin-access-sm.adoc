= Attribuez un accès d'administrateur SVM avec ONTAP System Manager
:hardbreaks:
:toclevels: 1
:allow-uri-read: 
:toclevels: 1
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
À partir d'ONTAP 9.19.1, les administrateurs du cluster ONTAP peuvent attribuer l'accès à System Manager à la personne qui gère le SVM, connue sous le nom d'administrateur SVM.

Ceci est particulièrement utile dans un environnement ONTAP mutualisé, où l'administrateur du cluster peut fournir un tableau de bord SVM à chaque locataire.

Les administrateurs de cluster peuvent également autoriser leurs administrateurs SVM à créer des volumes.



[[step-1-prepare-the-management-network-interface-for-the-svm-dashboard]]
== Étape 1 : Préparer l’interface réseau de gestion pour le tableau de bord SVM

Avant d'attribuer le tableau de bord SVM à votre administrateur SVM, assurez-vous que le SVM dispose d'une interface réseau de gestion. Si le SVM ne dispose pas d'une interface réseau, vous pouvez en créer une.

.Étapes
. Dans le Gestionnaire système, sélectionnez *Interfaces réseau > Vue d'ensemble* puis *Ajouter* pour créer une interface réseau.
. Sélectionnez *Storage VM management* comme type d'interface réseau.
. Spécifiez l'*adresse IP* et le *masque de sous-réseau* pour l'interface réseau de gestion SVM.




[[step-2-verify-the-vsadmin-user-account-configuration]]
== Étape 2 : Vérifier la configuration du compte utilisateur vsadmin

Lors de la création d'une SVM, ONTAP crée automatiquement un compte utilisateur vsadmin associé. Avant d'attribuer le tableau de bord de la SVM à l'administrateur, assurez-vous que le compte utilisateur vsadmin est protégé par un mot de passe et déverrouillé.

.Étapes
. Ouvrez le tableau de bord du Gestionnaire système en tant qu'administrateur du cluster.
. Sélectionnez *Cluster > Machines virtuelles de stockage*.
. Sélectionnez le SVM, sélectionnez *Paramètres* puis *Sécurité > Utilisateurs et rôles*.
. À côté de l'utilisateur *vsadmin*, sélectionnez image:icon_kabob.gif["Icône des options de menu"] et, si nécessaire, sélectionnez :
+
.. *Changer le mot de passe* pour mettre à jour le mot de passe vsadmin.
.. *Déverrouillez* pour vous assurer qu'il est utilisable.






[[step-3-allow-svm-administrators-to-add-volumes-optional]]
== Étape 3 : Autoriser les administrateurs SVM à ajouter des volumes (facultatif)

En tant qu'administrateur du cluster ONTAP, si vous souhaitez que vos administrateurs SVM puissent ajouter des volumes locaux dans leurs SVM, vous pouvez leur accorder l'autorisation.

.Étapes
. Ouvrez le tableau de bord du Gestionnaire système et connectez-vous en tant qu'administrateur du cluster.
. Sous *Cluster*, sélectionnez *Storage VMs*.
. Sélectionnez image:icon_kabob.gif["Icône des options de menu"] pour le SVM applicable.
. Sélectionnez *Modifier*.
. Sous *Allocation des ressources*, sélectionnez la case à cocher *Limiter la création de volumes aux niveaux locaux préférés*.
. Par défaut, tous les niveaux locaux sont inclus lorsque vous cochez la case à cocher. Vous pouvez supprimer des niveaux spécifiques si nécessaire. Si vous supprimez tous les niveaux locaux ou si vous laissez la case à cocher vide, les administrateurs SVM ne peuvent pas provisionner de volumes locaux.
. Sélectionnez *Enregistrer*.




[[step-4-assign-svm-administrator-access]]
== Étape 4 : Attribuer un accès administrateur SVM

Pour attribuer l'accès au tableau de bord SVM, fournissez l'adresse IP de gestion SVM et le mot de passe vsadmin à chaque administrateur SVM.

.Étapes
. Dans le Gestionnaire système, sélectionnez *Interfaces réseau > Vue d'ensemble* puis sélectionnez l'interface réseau *Management*.
. Sélectionnez l'adresse IP et communiquez-la, ainsi que le nom d'utilisateur et le mot de passe *vsadmin*, à l'administrateur SVM.
